# PRD — 회사 익명 게시판

## 1. 개요

회사 구성원만 접근 가능한 익명 게시판. 이메일 로그인으로 사내 구성원임을 인증하되, 게시글/댓글은 완전 익명으로 작성된다.

## 2. 기술 스택

| 영역 | 기술 |
|------|------|
| 프론트엔드 | Next.js 14 (App Router) |
| 백엔드 / DB | Supabase (PostgreSQL + Auth + RLS) |
| 스타일링 | Tailwind CSS |
| 배포 | Vercel |

## 3. 사용자

- **대상**: 회사 이메일 계정 소유자 (도메인 제한은 추후 설정)
- **익명성**: 로그인은 필요하나, 작성자 정보는 노출되지 않음

## 4. 핵심 기능

### 4.1 인증
- 이메일 + 비밀번호 회원가입 / 로그인
- Supabase Auth 사용
- 추후 특정 도메인(@company.com)만 허용하는 제한 추가 가능

### 4.2 게시글
- 제목 + 본문 작성
- 카테고리 선택 (예: 자유, 건의, 칭찬, 고민)
- 태그 입력 (선택)
- 목록 조회 (최신순 / 인기순)
- 상세 페이지 조회
- 작성자 정보 미표시 (익명)

### 4.3 댓글
- 게시글 하단에 익명 댓글 작성
- 댓글 목록 조회
- 댓글 수 표시

### 4.4 좋아요 / 반응
- 게시글에 좋아요 토글 (1인 1회)
- 좋아요 수 표시

### 4.5 UI / UX
- 라이트 / 다크 테마 전환
- 반응형 레이아웃 (모바일 대응)

### 4.6 디자인 시스템
- **레퍼런스**: Blind (teamblind.com) 스타일 — IT 회사 직원 대상 커뮤니티 감성
- **컬러**: 다크 네이비 계열 기본 팔레트, 포인트 컬러 Blue/Indigo
- **타이포그래피**: 시스템 폰트 + 가독성 중심, 숫자/코드는 모노스페이스
- **카드 레이아웃**: 게시글 카드에 댓글 수 / 좋아요 수 인라인 표시
- **헤더**: 로고 + 네비게이션 + 테마 토글 깔끔하게 정렬
- **빈 공간 활용**: 충분한 패딩, 호버 효과, 미묘한 경계선으로 계층 표현
- **뱃지/태그**: 카테고리를 색상 구분 뱃지로 표시 (자유=회색, 건의=파랑, 칭찬=초록, 고민=노랑)

## 5. 비기능 요구사항

- 익명성 보장: DB에서 작성자 user_id를 저장하되 API/UI에서 노출 금지
- RLS(Row Level Security)로 본인 글만 삭제 가능
- 비속어 필터링은 MVP 이후 고려

## 6. MVP 범위 (1차 출시)

- [ ] 이메일 회원가입 / 로그인
- [ ] 게시글 작성 / 목록 / 상세
- [ ] 카테고리 필터
- [ ] 댓글
- [ ] 좋아요
- [ ] 라이트 / 다크 테마

## 7. MVP 이후 고려 사항

- 회사 도메인 제한
- 관리자 페이지 (신고 처리, 게시글 삭제)
- 알림 기능
- 검색
- 태그 기반 필터링
