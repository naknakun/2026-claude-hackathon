## 테스트 전략

### 원칙

| 원칙 | 설명 |
|------|------|
| 격리 | 각 테스트는 독립적으로 실행 가능 (순서 의존 없음) |
| 실제 환경 | Supabase 실제 DB 사용 (mock 없음) |
| 정리 | 테스트가 생성한 데이터는 `afterEach`에서 삭제 |
| 빠른 피드백 | 핵심 흐름만 커버, 세부 UI 스타일 검증은 제외 |

### 테스트 계정

`.env.local`에 테스트 전용 계정 추가:
```
TEST_EMAIL=test@example.com
TEST_PASSWORD=testpassword123
```
> Supabase에서 미리 생성해둔 계정 사용 (테스트마다 가입하면 rate limit 발생)

### 실행 방법

```bash
# 개발 서버 실행 중 상태에서
npm run test:e2e          # headless 모드
npm run test:e2e:ui       # Playwright UI 모드 (디버깅용)
```

## 테스트 시나리오

### 1. Auth 흐름 (`auth.spec.ts`)

```
로그인 성공:
  1. /login 접속
  2. 테스트 계정 이메일/비밀번호 입력
  3. 로그인 버튼 클릭
  4. 홈(/) 리다이렉트 확인
  5. 헤더에 "로그아웃" 버튼 표시 확인

로그아웃:
  1. 로그인 상태에서 "로그아웃" 클릭
  2. 헤더에 "로그인" 버튼 표시 확인

미로그인 글쓰기 접근 차단:
  1. 비로그인 상태에서 /posts/new 직접 접속
  2. /login 리다이렉트 확인
```

### 2. Post 흐름 (`post.spec.ts`)

```
글 작성 → 목록 확인:
  1. 로그인
  2. "글쓰기" 클릭 → /posts/new 이동
  3. 카테고리 "건의" 선택
  4. 제목 "E2E 테스트 게시글" 입력
  5. 내용 입력
  6. "등록" 클릭
  7. 홈(/) 리다이렉트 확인
  8. 목록에 "E2E 테스트 게시글" 표시 확인

글 상세 → 삭제:
  1. 목록에서 해당 글 클릭
  2. 제목 표시 확인
  3. "삭제" 버튼 클릭
  4. 홈(/) 리다이렉트 확인
  5. 목록에서 해당 글 사라짐 확인
```

### 3. Search 흐름 (`search.spec.ts`)

```
키워드 검색:
  1. 로그인 후 고유 키워드 포함 게시글 작성
  2. 홈에서 검색창에 해당 키워드 입력 → Enter
  3. URL에 ?search=키워드 포함 확인
  4. 해당 게시글이 결과에 표시되는지 확인

빈 검색 결과:
  1. 존재하지 않는 키워드로 검색
  2. "검색 결과가 없습니다." 메시지 표시 확인

검색어 초기화:
  1. 검색 후 ✕ 버튼 클릭
  2. 전체 목록 복원 확인
```

### 4. Tag 흐름 (`tag.spec.ts`)

```
태그 작성 및 필터:
  1. 로그인 후 태그 "e2e-tag" 포함 게시글 작성
  2. 홈 목록에서 #e2e-tag 뱃지 표시 확인
  3. 상세 페이지에서 #e2e-tag 클릭
  4. URL에 ?tag=e2e-tag 포함 확인
  5. 해당 태그 게시글만 표시 확인
```

### 5. Comment 흐름 (`comment.spec.ts`)

```
댓글 작성 → 삭제:
  1. 로그인 후 게시글 상세 접속
  2. 댓글 입력 → "등록" 클릭
  3. 댓글 목록에 표시 확인
  4. 본인 댓글 "삭제" 클릭
  5. 댓글 목록에서 사라짐 확인
```

### 6. Like 흐름 (`like.spec.ts`)

```
좋아요 토글:
  1. 로그인 후 게시글 상세 접속
  2. 좋아요 버튼의 초기 숫자 확인
  3. 좋아요 버튼 클릭 → 숫자 +1 확인
  4. 다시 클릭 → 숫자 -1 확인 (취소)
```
