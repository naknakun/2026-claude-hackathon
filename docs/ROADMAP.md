# ROADMAP — 회사 익명 게시판

## 마일스톤 개요

```
Sprint 1 → 프로젝트 초기 세팅 & DB 스키마
Sprint 2 → 인증 (회원가입 / 로그인)
Sprint 3 → 게시글 CRUD + 카테고리
Sprint 4 → 댓글
Sprint 5 → 좋아요
Sprint 6 → 라이트 / 다크 테마 + 반응형 마무리
```

---

## Sprint 1 — 프로젝트 초기 세팅 & DB 스키마

**목표:** 개발 환경 구성 + Supabase 테이블 설계 완료
**완료 기준:** `npm run dev` 실행 시 Next.js 앱이 뜨고, Supabase에 테이블이 생성됨

### 태스크
- [ ] Next.js 14 프로젝트 생성 (App Router, Tailwind CSS)
- [ ] Supabase 프로젝트 생성 및 `.env.local` 연결
- [ ] DB 테이블 생성
  - `posts` (id, user_id, title, content, category, created_at)
  - `comments` (id, post_id, user_id, content, created_at)
  - `likes` (id, post_id, user_id, created_at)
- [ ] RLS 정책 설정 (읽기: 전체 허용 / 쓰기: 로그인 유저 / 삭제: 본인만)
- [ ] 폴더 구조 세팅 (`app/`, `components/`, `lib/`)

**의존성:** 없음

---

## Sprint 2 — 인증 (회원가입 / 로그인)

**목표:** 이메일로 가입하고 로그인/로그아웃 가능
**완료 기준:** 가입 → 로그인 → 헤더에 로그아웃 버튼 표시 흐름이 동작함

### 태스크
- [ ] Supabase Auth 클라이언트 설정 (`lib/supabase.ts`)
- [ ] 회원가입 페이지 (`/signup`) — 이메일 + 비밀번호
- [ ] 로그인 페이지 (`/login`) — 이메일 + 비밀번호
- [ ] 로그아웃 기능
- [ ] 로그인 상태에 따른 헤더 UI 분기
- [ ] 미로그인 시 글쓰기 페이지 접근 차단 (redirect)

**의존성:** Sprint 1

---

## Sprint 3 — 게시글 CRUD + 카테고리

**목표:** 글을 쓰고, 목록을 보고, 상세를 읽을 수 있음
**완료 기준:** 글 작성 → 목록에 노출 → 상세 페이지 이동 → 본인 글 삭제 가능

### 태스크
- [ ] 게시글 목록 페이지 (`/`) — 최신순, 카테고리 필터
- [ ] 게시글 상세 페이지 (`/posts/[id]`)
- [ ] 게시글 작성 페이지 (`/posts/new`) — 제목, 본문, 카테고리 선택
- [ ] 게시글 삭제 (본인 글만)
- [ ] 작성자 정보 미노출 (익명 표시)
- [ ] 카테고리 필터 UI (자유 / 건의 / 칭찬 / 고민)

**의존성:** Sprint 2

---

## Sprint 4 — 댓글

**목표:** 게시글 상세 페이지에서 익명 댓글 작성/조회
**완료 기준:** 댓글 작성 → 목록에 즉시 반영 → 댓글 수 표시

### 태스크
- [ ] 댓글 작성 폼 (게시글 상세 하단)
- [ ] 댓글 목록 조회
- [ ] 댓글 수 게시글 목록/상세에 표시
- [ ] 본인 댓글 삭제

**의존성:** Sprint 3

---

## Sprint 5 — 좋아요

**목표:** 게시글에 좋아요 토글 가능
**완료 기준:** 좋아요 클릭 → 수 증가/감소 → 새로고침 후에도 상태 유지

### 태스크
- [ ] 좋아요 버튼 UI (토글)
- [ ] 좋아요 수 표시
- [ ] 1인 1회 제한 (DB unique constraint)
- [ ] 인기순 정렬 (좋아요 많은 순)

**의존성:** Sprint 3

---

## Sprint 6 — 라이트 / 다크 테마 + 반응형 마무리

**목표:** 테마 전환 + 모바일 레이아웃 완성
**완료 기준:** 테마 토글 클릭 시 전환되고 새로고침 후에도 유지, 모바일에서 레이아웃 깨짐 없음

### 태스크
- [ ] Tailwind dark mode 설정 (`class` 전략)
- [ ] 테마 토글 버튼 (헤더)
- [ ] localStorage로 테마 상태 유지
- [ ] 전체 페이지 모바일 반응형 점검
- [ ] 빈 상태 UI (글 없을 때 안내 메시지)

**의존성:** Sprint 4, 5

---

---

## Sprint 7 — UI 리디자인 (Blind 스타일)

**목표:** 기본 Tailwind 디자인을 IT 커뮤니티 감성으로 전면 개편
**완료 기준:** 블라인드와 유사한 카드형 레이아웃, 카테고리 색상 뱃지, 깔끔한 헤더, 다크모드 완성도 향상

### 태스크
- [ ] 글로벌 컬러 팔레트 정의 (네이비/인디고 포인트 컬러)
- [ ] 헤더 리디자인 (로고 + 네비 + 테마 토글 정렬)
- [ ] 게시글 카드 리디자인 (카테고리 색상 뱃지, 댓글 수 / 좋아요 수 인라인)
- [ ] 카테고리 필터 탭 스타일로 변경
- [ ] 로그인 / 회원가입 페이지 리디자인
- [ ] 글쓰기 페이지 리디자인
- [ ] 게시글 상세 페이지 리디자인 (댓글, 좋아요 포함)
- [ ] 다크모드 전체 컴포넌트 완성도 향상

**의존성:** Sprint 6

---

## Sprint 8 — 게시글 검색 + 태그 필터링

**목표:** 키워드 검색과 태그 기반 탐색 기능 추가
**완료 기준:** 검색창에 키워드 입력 시 제목/내용 필터링, 태그 클릭 시 관련 게시글 목록 표시

### 태스크

#### DB
- [ ] `posts` 테이블에 `tags text[]` 컬럼 추가 (Supabase SQL Editor)
- [ ] 태그 검색용 GIN 인덱스 추가

#### 기능
- [ ] 검색 입력창 UI (목록 상단)
- [ ] 제목 + 내용 키워드 검색 (`ilike` 기반, URL: `/?search=키워드`)
- [ ] 태그 입력 UI (글쓰기 페이지 — 쉼표 구분 입력)
- [ ] 태그 뱃지 표시 (게시글 카드 + 상세 페이지)
- [ ] 태그 클릭 시 필터링 (`/?tag=태그명`)
- [ ] 검색 + 카테고리 + 태그 복합 필터 동작

#### 테스트
- [ ] Vitest + React Testing Library 세팅
- [ ] 검색 쿼리 유틸 함수 단위 테스트
- [ ] 검색 컴포넌트 렌더링 테스트
- [ ] Playwright E2E 테스트 (검색 → 결과 확인, 태그 필터 흐름)

**의존성:** Sprint 7

---

## Sprint 9 — Playwright E2E 테스트

**목표:** 핵심 사용자 흐름을 자동화된 E2E 테스트로 검증
**완료 기준:** 인증 / 게시글 / 검색 / 태그 / 댓글 / 좋아요 흐름 모두 통과

### 태스크
- [ ] Playwright 설치 및 설정
- [ ] 테스트 환경 설정 (baseURL, 테스트 계정)
- [ ] Auth 흐름: 회원가입 → 로그인 → 로그아웃
- [ ] Post 흐름: 글 작성 → 목록 확인 → 상세 확인 → 삭제
- [ ] Search 흐름: 키워드 검색 → 결과 확인 → 빈 결과 확인
- [ ] Tag 흐름: 태그 글 작성 → 태그 클릭 → 필터 확인
- [ ] Comment 흐름: 댓글 작성 → 확인 → 삭제
- [ ] Like 흐름: 좋아요 → 수 증가 → 취소 → 수 감소

**의존성:** Sprint 8
